1. 平台简介

GyneCold AI 是一个面向转录组差异表达分析的智能研究平台。平台由三部分组成：

  • GyneCold AI 前端网站 — 你当前看到的界面，部署在 GitHub Pages 上，提供注册登录和对话入口
  • AI Agent 后端 — 大语言模型（DeepSeek Chat）驱动的分析助手，理解自然语言并自动调用分析工具
  • autoSX 分析引擎 — 后端的生物信息学分析管线，实现 limma 差异表达检验、火山图/热图/PCA 可视化

你只需要用中文告诉 AI 你想做什么，Agent 会自动完成文件上传、参数配置、任务提交、结果获取等全部流程。

2. 注册与登录

所有分析功能需要登录后才能使用。

1

访问首页

打开 GyneCold AI 首页，点击右上角的 "登录" 按钮，或中间的 "开始研究" 按钮。

2

注册账号

如果是第一次使用，点击 "立即注册"。填写姓名（可选）、用户名和密码即可完成注册。所有数据仅保存在你当前浏览器的本地存储中，不会上传到任何服务器。

3

登录

已有账号则直接输入用户名和密码登录。登录后自动跳转到分析聊天页面。

💡 提示：注册信息保存在浏览器 localStorage 中。如果更换设备或清除浏览器数据，需要重新注册。建议使用固定的用户名和密码。你也可以点击 "跳过登录，直接使用" 以访客身份体验。

3. 使用 AI 分析助手

登录后进入全屏聊天界面。这是你和 AI 分析助手的主要交互页面。

3.1 界面布局

  • 左侧边栏：显示当前用户名、新建对话按钮、快捷问题
  • 中央聊天区：显示对话历史，AI 回复以打字机效果逐字输出
  • 底部输入区：文本输入框、文件上传按钮、发送按钮

3.2 发送消息

在底部输入框中输入你的分析需求，按 Enter 发送（Shift+Enter 换行）。AI 会理解你的意图并自动执行相应操作。

3.3 快捷问题

侧边栏提供了 4 个快捷问题按钮，点击即可快速发起常见分析任务：

  • 📊 差异表达分析 — 开始一个标准的差异表达分析流程
  • 🔍 查看分析模块 — 查询当前平台上有哪些可用的分析模块
  • 📤 上传文件分析 — 上传文件并开始分析
  • ⚙️ 参数配置 — 设置 P 值、log2FC 等参数

3.4 打字机效果

AI 的回复会以打字机效果逐字显示，让你看到思考过程。对于较长的分析报告，可以耐心等待完整输出。

3.5 新建对话

点击侧边栏的 "+ 新建对话" 按钮可以清空当前对话历史，开始一个新的分析会话。

4. 上传文件

进行差异表达分析需要提供两个关键文件：

  • 表达矩阵 CSV — 行为基因（需包含 gene_id 列），列为样本，值为表达量
  • 样本分组 CSV — 包含 sample_id 和 group 两列，仅允许两组比较
1

点击上传按钮

在聊天输入框左侧，点击 📎 上传文件 按钮，选择你要上传的 CSV 文件。

2

预览文件

上传后会在输入框上方显示文件缩略图或文件名。点击 ✕ 可以移除不需要的文件。

3

告诉 AI 文件位置

在输入框中描述你的文件路径，例如：

表达矩阵在 D:\data\expression.csv，分组文件在 D:\data\groups.csv，帮我做差异表达分析

AI 会自动读取并上传文件到分析引擎。

⚠️ 注意：文件需要存放在 AI Agent 服务器可以访问的路径下。如果你使用的是本地部署的 Agent，文件路径需要是服务器上的绝对路径。当前演示环境下，文件路径以 Docker 容器内的路径为准。

5. 配置分析参数

RNA-seq 差异表达分析支持以下可调参数，你可以通过自然语言告诉 AI 来设置：

5.1 参数一览

参数 默认值 说明
p_value_threshold 0.05 显著性水平阈值，越小越严格
log2fc_threshold 1.0 差异倍数（log2）阈值，绝对值需大于此值
test_method limma 统计方法：limma / ttest / mannwhitney
normalization cpm 标准化：none / cpm / tpm / quantile
log_transform auto 是否 log2 转换：auto / yes / no
min_samples_expr 0 低表达基因过滤（N 个样本中表达量>0）

5.2 通过对话设置参数示例

👤 "用 limma 方法跑差异分析，p 值设为 0.01，log2FC 设为 1.5"
🤖 好的，已配置参数：limma 方法，p<0.01，|log2FC|>1.5。分析进行中...

👤 "标准化用 quantile，不过滤低表达基因"
🤖 已更新参数：标准化方法改为 quantile，min_samples_expr=0（不过滤）。

6. 查看与解读结果

分析完成后，AI 会以自然语言总结分析结果，并提供可视化图表和数据的访问链接。

6.1 AI 总结报告

分析完成后，AI 会给出类似这样的总结：

✅ 差异表达分析完成！
📊 结果概览：
   • 检测基因总数：21,555
   • 显著差异基因：127 个（padj<0.05, |log2FC|>1）
   • 上调基因：82 个 | 下调基因：45 个
   • 统计方法：limma | 标准化：CPM

📈 可视化图表：
   • 火山图 → 访问链接
   • 热图   → 访问链接
   • PCA 图 → 访问链接

📋 数据表格：
   • 差异基因列表 → 访问链接
   • 统计摘要     → 访问链接

6.2 查看图表

火山图：展示基因的差异表达分布。横轴为 log2FC（差异倍数），纵轴为 -log10(padj)（显著性）。左上和右上区域为显著差异基因。

  • 红色/绿色点 = 显著上调/下调的基因（padj < 阈值）
  • 灰色点 = 不显著基因
  • 被标注的基因是变化最显著的候选基因

热图：展示显著差异基因在样本间的表达模式。行 = 基因，列 = 样本，颜色表示 Z-score 标准化后的相对表达水平。

  • 红色 = 高表达，蓝色 = 低表达
  • 顶部样本标签颜色区分不同实验组
  • 横向聚类反映基因表达模式的相似性

PCA 图：主成分分析降维可视化，展示样本间的全局相似性。

  • 每个点代表一个样本，颜色代表分组
  • 椭圆表示 95% 置信区间
  • 横纵轴标注方差解释率（PC1/PC2）
  • 两组样本在 PCA 空间分离越好，表示差异越明显

6.3 下载数据

点击 AI 提供的链接可以查看完整图表或下载差异基因表格。

  • 差异基因表格：包含 gene_id、gene_symbol、log2FC、p_value、padj、significant 等列
  • 统计摘要：total_genes / up_regulated / down_regulated / significant 计数
💡 解读要点：关注 log2FC 的绝对值大小（效应量）和 padj 值（统计显著性）。结合火山图的可视化分布和热图的表达模式，综合判断最值得深入研究的候选基因。

8. 常见问题

Q: 为什么 AI 回复 "无法连接到服务"？

AI Agent 后端需要独立启动。如果你看到这个错误，说明后端服务未运行或网络不通。请确认 Agent 服务器已启动并可访问。

Q: 注册后换浏览器数据还在吗？

不在了。注册信息保存在当前浏览器的 localStorage 中。换设备或清除浏览器缓存后需要重新注册。建议使用固定用户名。

Q: 分析任务能跑多久？

取决于数据量大小。对于 2 万个基因、十几个样本的典型 RNA-seq 数据，通常在 1-3 分钟内完成。Agent 会自动轮询进度。

Q: 支持哪些输入文件格式？

目前支持 CSV 格式的表达矩阵和分组文件。表达矩阵要求行=基因、列=样本，需要包含 gene_id 列。分组文件需要包含 sample_id 和 group 两列。

Q: 能分析几组比较？

目前仅支持两组比较（如实验组 vs 对照组）。样本分组文件中的 group 列只能有两个不同的分组值。

Q: 首页的 DNA 双螺旋动画很吃性能吗？

不会。Three.js 动画使用 WebGL 硬件加速，对现代浏览器和设备的性能影响极小。如果感觉卡顿，滚动页面后动画会自动暂停渲染循环。
